El principal cuello de botella en tu código es la cantidad de veces que calculas el SSIM (Structural Similarity Index) entre pares de imágenes. Actualmente, estás calculando el SSIM entre todas las imágenes para cada imagen individual, lo que resulta en un algoritmo de complejidad cuadrática O(n^2). Esto puede ser extremadamente lento para una gran cantidad de imágenes.

Si necesitas un método más rápido para calcular la similitud entre dos imágenes que el SSIM, puedes considerar utilizar otros enfoques como la diferencia estructural (Structural Difference, SD), la diferencia absoluta (Mean Absolute Error, MAE), la diferencia cuadrática (Mean Squared Error, MSE), o histogramas de color.

Aquí te muestro cómo podrías implementar algunos de estos métodos en tu código:

from skimage import exposure

def calculate_hist_similarity(image1, image2):
    hist1, _ = exposure.histogram(image1)
    hist2, _ = exposure.histogram(image2)
    return np.sum(np.minimum(hist1, hist2)) / np.sum(np.maximum(hist1, hist2))

# En el bucle de cálculo de similitud:
score = calculate_hist_similarity(archivo_i[k], archivo_j[k])
